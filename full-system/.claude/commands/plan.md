Generate the weekly meal plan.

## Step 0 — Week Turnover (do this FIRST, in this exact order)

### 0.1 Deduct last week's meals
Find the most recent dated folder in `weekly/`. Its `meal-plan.md` is **last week's plan**. Check the dated notes at the top of the inventory files:

- If they already show last week was deducted → done, move to 0.2.
- Otherwise ask: "Which of last week's dinners actually got cooked? (default: all of them)" — then deduct their ingredients from the inventory files (same logic as `/week-used`) and add a dated note to each updated file: `Updated YYYY-MM-DD — deducted week of [dates]`.

### 0.2 Get the receipts (SEPARATE question — never skip, never bundle with 0.1)
Check the inventory files' dated notes for a receipt added since the last deduction. If there isn't one, ask and **wait for an answer**:
> "Do you have receipts from your shopping trip(s) last week? Paste them, drop a photo, or say 'no receipts' — I won't plan until I hear back."

- Receipts provided → add items to the right inventory files (same logic as `/inventory-add`) and note: `Updated YYYY-MM-DD — added receipt ([store], [receipt date])`
- "No receipts" → note it so this question isn't re-asked.

The deduction being done does NOT mean the receipts are done. They are independent checks.

### 0.3 Set up this week's folder
Create `weekly/[YYYY-MM-DD]/` (date = this coming Monday) by copying `weekly/template/`. If the user hasn't filled in `schedule.md`, ask:
> "What's on the calendar this week? Evening events, busy nights — and any camping days?"

If the week includes camping → read `family/camping-mode.md` and apply camping mode to those days.

## Step 1 — Gather inputs

Read: `family/family-preferences.md`, `family/household-rules.md`, `inventory/freezer.md`, `inventory/pantry.md`, `inventory/fridge.md`, `planning/needs-this-week.md`, this week's `schedule.md`, and `stores/this_week.md` if it exists.

**Candidate meal pool = three equal sources.** Do not let the recipe library crowd out plain meals:
1. AnyList recipes — use the AnyList MCP: list recipes, list recent meal-plan events (avoid repeats from the last 2–3 weeks), and check recipe collections (slow cooker, grill, etc.) to match meal types to the week
2. **"Meals the Family Loves"** from `family/household-rules.md` — plain dinners that don't need a recipe. Most weeks should include 1–3 of these.
3. Deal-driven ideas from `stores/this_week.md`

**Weather:** search "[user's city] weather forecast next 7 days" (ask for the city once if unknown). Hot days = no oven meals.

## Step 2 — Apply the planning rules

Apply every rule from `family/household-rules.md`. The usual set:
- Prioritize freezer proteins before buying new meat
- No red meat back-to-back nights; protein every dinner
- Easy/slow-cooker meals on nights with evening events
- No oven meals on hot days
- No repeats from the prior 2 weeks (check AnyList meal-plan events + `weekly/` archive folders)
- Every dinner gets a complete plate: vegetable + starch (note "included" if the recipe covers it)
- Plan 1–2 meals around sale items and 1–2 around what's already in inventory
- Camping days follow `family/camping-mode.md`

## Step 3 — Generate the plan

Format:
```
## Quick View
Mon: [Meal] | [Sides]
Tue: [Meal] | [Sides]
...

## Lunch Plan
[per person — leftovers first, then the rotation from family-preferences.md]

## Thaw Schedule
[night-before thaw reminders]

## Protein Rotation Check
[verify no back-to-back red meat]
```

Then a **Recipe Notes** section: for each dinner — where the recipe lives (AnyList name, or "family favorite / no recipe needed"), key ingredients to buy, what's already in inventory.

## Step 4 — Write and confirm

Write the plan to `weekly/[YYYY-MM-DD]/meal-plan.md`. Line 1: today's date + "Generated by Roux". Line 2: `Week of [Mon date] – [Sun date]` — **required**; the turnover and `/week-used` guards read it.

Confirm: "Plan written for [dates]. Ready to run `/shop` whenever you are."
